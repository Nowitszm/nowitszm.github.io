iTwin.js 理论基础 - 编码规范

2021-05-20 computer graphics 1.6k words
iTwin Platform

下文是我们希望所有iTwin.js开发者遵循的TypeScript编码准则。在默认情况下，这些准则是通过我们的TSLint配置文件（tslint.json）执行

"extends": "./node_modules/@bentley/build-tools/tsconfig-base.json"

Name

  • 对类、枚举值使用PascalCase（帕斯卡）命名。（eg:"IModelApp"）
  • 对函数、属性和局部变量使用camelCase（驼峰）命名。(eg:"getFirstViewDefinitionId")
  • 不要使用前缀来作为接口名称。（eg:"imjs"）
  • 对私有属性使用前缀。（eg:"_onIModelSelect"）
  • 在命名过程中使用全部单词，尽量少用缩略语。

File

  • TypeScript文件使用.ts作为文件扩展名。
  • TypeScript的文件名应该是PascalCase命名。

Type

  • 在不需要多个组件共享使用类/函数时，不要导出。
  • 不要在全局命名空间中引入新的类型。
  • 在一个ts文件中，首先要进行类的定义。

Do not use null

  • 尽可能不要使用null或者undefined。

Operators

  • 尽可能使用===/!==运算符而不是==/!=。

Strings

  • 对字符串使用双引号。

General Constructs

  • 始终使用分号;我们的TSLint规则强制执行这一点，在VSCode中会提示出错，单不影响使用。
  • 使用 {} 替代 new Object()。
  • 使用 [] 替代 new Array()。

Make judicious use of vertical screen space

  • 尽量不要使用多余空行。例如，一个函数的第一行不应该是一个空行。

  • 尽量不要把导入语句中的每个导入放在单独的一行。如果你使用Visual Studio Code作为你的编辑器，使用TypeScript Import Sorter扩展的默认设置来自动格式化导入语句。

  • 尽量不要使用巧妙的/漂亮的多行注释块来分隔代码部分。

  • 尽量不要让只有一条函数语句的函数放在同一行。

    //No
    public middle(): number { return this.minimum + ((this.maximum - this.minimum) / 2.0); }

    //Yes
    public middle(): number {
       return this.minimum + ((this.maximum - this.minimum) / 2.0);
    }

  • if语句或循环的主体应该在一个单独的行上，即使主体只包含一行代码。

    // No
    if (meow) return "cat";

    // Yes
    if (meow)
     return "cat";

  • 结尾的大括号应该是一个空行

    // No 
    if (minimum > maximum) {
      const temp = minimum;
      minimum = maximum;
      maximum = temp;
    }
    return maximum - minimum;
    // Yes
    if (minimum > maximum) {
      const temp = minimum;
      minimum = maximum;
      maximum = temp;
    }

    return maximum - minimum;

  • 尽量不要省略单行代码的大括号{}

    // No (closing brace wastes a line) !!!
    if (meow) {
      return "cat";
    }
    // Correct (no braces) !!!
    if (meow)
      return "cat";

Style

  • 在匿名函数表达式上使用箭头函数。

  • 不要使用var，而是尽可能使用const或let。

  • 每个变量语句使用一个声明（eg:let x = 1; let y = 2）。

  • {}结构的周围不应有任何空白。,``:``;这些结构体中的运算符后面都有一个空格。

    for (let i = 0, n = str.length; i < 10; ++i) { }
    if (x < 10) { }
    public calculate(x: number, y: string): void { . . . }

  • 每次缩进使用两个空格，不要使用制表符。

  • 建议在编辑其中打开tslint。

return

  • 当返回语句有返回值时，不要使用()。

    return ("Hello World!"); // bad

    return "Hello World!"; // good

  • 某些编程流派认为，每个方法都应该只有一个返回语句，iTwin.js建议当你认为没有理由继续执行下去就应该立刻返回。

    // bad!!
    public getFirstUser(): Person | undefined {
      let firstUser?: Person;
      if (this.hasUsers()) {
        if (!this.usersValidated()) {
          if (this.validateUsers())
            firstUser = this.getUser(0);
        } else {
            firstUser = this.getUser(0);
        }
      }
      return firstUser;
    }

    // ok!!
    public getFirstUser(): Person | undefined {
      if (!this.hasUsers())
        return undefined;

      if (!this.usersValidated() && !this.validateUsers())
          return undefined;

      return return this.getUser(0);
    }

    // best!!! For a simple case like this that is deciding between 2 return values
    public getFirstUser(): Person | undefined {
      const userInvalid = !this.hasUsers() || (!this.usersValidated() && !this.validateUsers());
      return userInvalid ? undefined : this.getUser(0);
    }

  • 尽可能明确定义一个返回类型，这也是TypeScript的编程理念。

    // bad!! No return type specified
    public getOwner(name: string) {
      if (this.isReady)
        return this.widget.getOwner(); // is this a Person???
      ...
      return new Person(name);
    }

    // good!!
    public getOwner(name: string): Person {
      if (this.isReady)
        return this.widget.getOwner(); // if this is not a Person, compile error
      ...
      return new Person(name);
    }

  • 当调用方法时，明确指定返回类型。在异步方法调用的情况下，这种做法可以防止省略await关键字。

    // bad!! no return type specified for async call, and missing await is not caught by the compiler
    const iModels = iModelHub.getIModels(projectId);

    // good!! omitting the await would be caught by the compiler as a type mismatch
    const iModel: IModel[] = await iModelHub.getIModels(projectId);

Getters and Setters

  • getter的常见用法是有一个私有成员，该成员在类中是私有读/写的，但对公共API是只读的。

    class Person {
      private _name: string;
      public get name(): string { return _name; } // read-only to public API, so no setter provided
    }

  • 如果getter提供的值是在构造函数中建立的，并且永远不会被改变，可以参照下文。

    class Person {
       constructor(public readonly name: string) { }
    }

  • getters和setters的另一个常见用法是，当你想给一个公共成员，但你实际上并不以这种方式存储数据。

    class PersonName {
      constructor (public firstName: string, public lastName: string) { }
      public get fullName(): string { return this.firstName + " " + this.lastName; }
      public set fullName(name: string): void {
        const names: string[] = name.split(" ");
        this.firstName = names[0] || "";
        this.lastName = names[1] || "";
      }
    }

use “?:” syntax vs. “ | undefined”

  • 当声明成员变量或函数参数时，使用TypeScript的"?: "语法，而不是为可以未定义的变量添加"| undefined"。
    class Role {
      public name: string;
      public description: string | undefined; // Wrong !!
    }

    class Role {
      public name: string;
      public description?: string; // Correct !!
    }

Don’t repeat type names unnecessarily

  • TypeScript是向JavaScript增强类型的，编译器会自动推断类型，因此没有必要在每个成员或变量的声明中装饰其类型，适当的添加类型。
  • 如果一个函数超过一行，最好总是包括它的返回类型，以确保没有返回路径有一个意外的类型。
    let width: number = 7.3; // useless type declaration
    public isReady: boolean = false; // useless type declaration
    public readonly origin: Point3d = new Point3d(); // useless type declaration

    let width = 7.3; // correct
    public isReady = false; // correct
    public readonly origin = new Point3d(); // correct
    const upVector: Vector3d = rMatrix.getRow(1); // good, helps readability. Not strictly necessary.

Asynchronous Programming

  • 使用Promise
  • 使用return Promise.reject(new MyError())而不是解析为错误状态 。传入 Promise.reject 的对象应该是 Error 的子类。
  • 使用async/await比.then()更好

Copyright notice

  • 每个.ts文件的第一行
    /*---------------------------------------------------------------------------------------------
    * Copyright (c) Bentley Systems, Incorporated. All rights reserved.
    * See LICENSE.md in the project root for license terms and full copyright notice.
    *--------------------------------------------------------------------------------------------*/

Source Code Editor

  • 推荐使用Visual Studio Code
  • 建议为Visual Studio Code配置ESLint扩展，并使用我们的 @bentley/eslint-plugin

React Function Components

有多种方式定义React函数组件：

  1. const Xyz: React.FunctionComponent<XyzProps> = (props) => {...}
  2. const Xyz: React.FC<XyzProps> = (props) => {...}
  3. function Xyz(props: XyzProps) {...}
    //good
    export function Xyz(props: XyzProps) {
      return <div>Hello World!</div>;
    }
