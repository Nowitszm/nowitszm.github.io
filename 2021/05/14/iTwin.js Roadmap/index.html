iTwin.js Roadmap

2021-05-14 1.5k words

本文概述了iTwin.js平台的开发路径，对功能开发的优先级进行简单说明，它会随着iTwin.js每个版本的更新而更新。

主题

我们将使用以下主题（更具实际的需求不断改变）来推动iTwin.js的发展：

  • 作为基础设施数字孪生的开放平台，将 BIM 模型、GIS 数据、实景数据、IT 和 OT 联合在一起，实现无缝的用户体验。
  • 分离对云供应商、操作系统和浏览器之间的依赖。
  • 高性能、易扩展性、安全性和稳定性是iTwin.js的核心。
  • 让使用 iTwin.js 进行开发尽可能简单、高效。
  • 尽可能遵守开源标准和规范。
  • 构建一个创新和企业的生态系统。

路线图

选中的框表示在最新版本中完成的功能。如果未选中的框，则表示该功能的工作正在进行中或暂定于将来发布。本文档的各个部分功能的会随时间的推移而变化，项目通常按优先级排序（注意：优先级顺序并不一定代表完成顺序，因为优先级较低但简单的功能可能会很快完成）。如果某个项目的优先级低于您的预期，则可以通过提交 PR 来提出自己的意见和建议。点击此连接提交PR。

可视化

  • √支持硬件抗锯齿
  • √点云和地形的主题显示
  • ×支持压缩的纹理格式
  • √粒子效果
  • √屏幕空间效果
  • ×大型场景动画（例如，行人模拟、天气动画、灾难模拟）
  • ×支持大气层渲染效果

### 拓展
  • ×支持与UI技术无关的”内置”和运行时扩展（即不依赖于React，Angular，Vue.js等）
  • ×支持添加工具和最小的 UI 功能
  • ×创建用于构建扩展的示例和模板
  • ×创建快速简便的支持，以便在扩展的托管应用程序中进行创建 - 例如，iTwin Viewer

地理信息系统显示

  • √扩展基础地图的支持（AzureMaps、MapBox等），基础地图的透明度可以独立于图层来控制。
  • √添加对显示连接到实时 WMS、WMTS、AzureMaps、MapBox 或 ArcGIS REST API 服务器的栅格地图图层的支持。可以独立控制地图图层的可见性和透明度。图层可以作为背景附加到 BIM 几何图形下方，也可以作为叠加层附加到 BIM 几何图形上方。

微软集成

  • ×提供用于直接在 Microsoft Teams 中托管 iTwin.js 应用程序的 API。

ET/IT/OT 集成

  • ×为iModels中的工程项目、企业系统和实时物联网传感器、相机、控制器、设备和处理器之间的接口提供工作实例和模板。
  • ×创建用于增强与具有 IOT 链接元素的工程设计工具的连接器同步的数据的工具。
  • ×增强常见物联网模式和查询引擎的BIS模式。

代理部署

  • ×提供通过 Kubernetes 部署的代理的工作示例和模板。
  • ×对 iModel 执行由 iModelHub 中的相关事件触发的 iModel 操作。

iTwin Viewer

  • √创建功能齐全的 iTwin Web 查看应用程序平台，可部署在任何云基础架构上。
  • ×提供必要的基础结构，以便在任何基于 iTwin Viewer的解决方案中托管扩展。

点云数据

  • ×支持大规模点云基于时间的差异可视化。

实景数据

  • √支持为大型实景数据实施网格化提供线性转换。

地理照片和全景图

  • ×改进服务器端对照片和全景图标签的提取。
  • ×将地质照片包纳入可从任何iTwin浏览器应用程序加载的扩展程序。

地理定位（普通）

  • √提供一种方法来访问 iModel 和垂直的地理位置 gcs 定义。
  • ×根据 orbit gt 实现实现前端重新投影。
  • ×实现新的地理坐标服务的客户端。
  • ×从新的地理坐标服务客户端获取大地水准面分离。
  • ×在 iTwin 工作区支持中完成系统网格文件。

iModel Transformations

  • √创建示例和模板，说明 iModel 到 iModel 转换和同步的用法。

iTwin.js 连接器框架

  • √支持使用 iTwin.js 后端创建多进程连接器。一个进程与源应用程序 API 链接以读取源应用程序文件。然后，它通过IPC与另一个iTwin.js进程进行通信，以更新iModel。

iModel 编辑应用程序

  • √支持通过 Electron 在桌面上进行交互式 iModel 创建、编辑和扩充。

移动设备支持

  • ×最大限度地降低移动浏览器的内存、功耗和带宽要求。
  • √提高触摸和小屏幕外形的可用性。
  • ×支持完全离线和偶尔连接的平板电脑和手机（iOS和Android）。

非图形数据显示

  • √通过支持的ECSchema版本过滤演示规则和规则集。
  • √在以下情况下，自动更新所有规则驱动的组件中的数据：
    • √源iModel数据的更改
    • √演示文稿规则更改
    • √表示规则集变量更改

属性相关显示

  • √嵌套属性分组。
  • √选择目标元素的可单击导航属性。
  • ×更好地支持复合属性（数组、结构、点）。
  • ×支持关系属性。
  • ×支持临时属性。

用户界面

  • √扩展基于面板的 AppUi 的功能。
  • ×改善移动设备上的触摸支持和用户体验。
  • √桌面 Electron 应用程序的多窗口支持。
  • ×增强辅助功能和键盘导航。
  • ×增强 UiItemProvider 以支持：
    • ×更多文字编辑器，尤其是读/写工作流所需的文字编辑器
    • ×异步列表填充
  • √提供 API 以允许应用、扩展和程序包向”设置”阶段提供设置。

与外部系统的接口

  • √为建立其他文件格式（如glTF、OBJ、USD）的导出器提供基础功能。
  • √展开图形提取的筛选和质量控制选项。

开发人员体验

  • √将 iTwin.js和相关存储库迁移到 GitHub。
